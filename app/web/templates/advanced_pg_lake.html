pg_lake Coming Soon

PostgreSQL に Apache Iceberg とデータレイクアクセス機能を追加する Snowflake 製エクステンション。 pg_lake は Snowflake Labs が開発するオープンソースプロジェクトです。

現在利用不可: pg_lake は pg_extension_base が shared_preload_libraries に登録されている必要がありますが、 現在の Snowflake Postgres 環境ではまだ設定されていません。Snowflake 側の対応待ちです。
概要
  • CREATE TABLE ... USING iceberg で Iceberg テーブルを作成
  • 通常の INSERT / UPDATE / DELETE がそのまま動作
  • S3 上の Parquet / CSV / JSON を Foreign Table で直接参照
  • COPY TO/FROM 's3://...' でデータレイク読み書き
  • パーティショニング: day(), month(), bucket()

GitHub Repository

SQL サンプル（将来利用可能）
-- Iceberg テーブル作成
CREATE TABLE events (
  event_time timestamptz,
  user_id int,
  data text
) USING iceberg
  WITH (partition_by = 'day(event_time)');

-- 通常の DML
INSERT INTO events VALUES (now(), 1, 'hello');

-- S3 へエクスポート
COPY events TO 's3://bucket/events.parquet';

-- S3 の Parquet を外部テーブルとして参照
CREATE FOREIGN TABLE ext_data ()
  SERVER pg_lake
  OPTIONS (path 's3://bucket/*.parquet');
