pg_hint_plan Demo

統計情報の劣化により非効率な実行計画が選ばれるケースを再現し、ヒントで修正する体験をします。 pg_hint_plan は SQL コメントで実行計画を制御する PostgreSQL エクステンションです。
開発: NTT OSS Center DBMS Development and Support Team / ライセンス: PostgreSQL License

Setup SQL

    









    
シナリオ解説

    

        
    
            
  1. hint_demo.orders テーブルに 500,000 行を INSERT
                
    status = shipped / processing / delivered / cancelled の4種（'rush' は存在しない）
    2. 
            
  2. ANALYZE で統計情報を取得 → プランナーは 'rush' を知らない
    3. 
            
  3. 追加で status='rush' の行を 200,000 行 INSERT（ANALYZE しない）
    4. 
            
  4. この時点で rush は実際 200,000 行（全体の28%） だが、統計にはゼロ
    5. 
            
  5. プランナーは「rush は存在しない or 極少数」と誤認 → Index Scan を選択 → 大量のランダム I/O
    6. 
            
  6. /*+ SeqScan(orders) */ ヒントで Seq Scan を強制 → 大量行にはシーケンシャルが効率的
    7. 
        

        

            Note: このデモでは max_parallel_workers_per_gather = 0 でパラレルクエリを無効化し、
            単一プロセスでの Index Scan vs Seq Scan の違いを明確にします。

            前提条件: session_preload_libraries または shared_preload_libraries に
            pg_hint_plan が設定されている必要があります。Setup で自動チェックします。
            
Snowflake Postgres: 現時点では session_preload_libraries に pg_hint_plan を設定できないため、
            このデモは動作しません（Snowflake への機能リクエストが必要）。
        

    







    
Step 1: 統計情報の確認

    

        
ANALYZE 後の古い統計 vs 実際の行数を比較します。

        
-- 実際の行数を確認
SELECT status, COUNT(*) AS actual_count
FROM hint_demo.orders GROUP BY status ORDER BY actual_count DESC;

-- プランナーの統計情報（pg_stats）
SELECT most_common_vals, most_common_freqs
FROM pg_stats
WHERE schemaname = 'hint_demo' AND tablename = 'orders' AND attname = 'status';

        
        

    







    
Step 2: 実行計画の比較（ヒントなし vs あり）

    

        
同じクエリをヒントなし・ありで EXPLAIN ANALYZE し、実行計画と所要時間を比較します。

           両方とも /advanced/hint-query エンドポイントで同一セッション内に LOAD + SET を実行します。

        
        

            

                

                    

                        ヒントなし（プランナー任せ）
                    

                    

                        
EXPLAIN ANALYZE
SELECT count(*), avg(amount)
FROM hint_demo.orders
WHERE status = 'rush';

                        

                        

                    

                

            

            

                

                    

                        ヒントあり（SeqScan 強制）
                    

                    

                        
/*+ SeqScan(orders) */
EXPLAIN ANALYZE
SELECT count(*), avg(amount)
FROM hint_demo.orders
WHERE status = 'rush';

                        

                        

                    

                

            

        

    







    
Step 3: ANALYZE で統計を更新

    

        
ANALYZE を実行して統計情報を最新化すると、プランナーが正しい計画を選ぶようになります。

        
-- 統計情報を更新
ANALYZE hint_demo.orders;

-- ヒントなしでも正しい計画を選ぶか確認
EXPLAIN ANALYZE
SELECT count(*), avg(amount)
FROM hint_demo.orders
WHERE status = 'rush';

        
        

    







    Benchmark 連携:
    ヒントあり/なしの SQL を Scripts & Scenarios に登録し、
    Benchmark のカスタムシナリオで実行すると、TPS やレイテンシの違いを数値で比較できます。
